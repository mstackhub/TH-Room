# คู่มือการติดตั้งระบบ (System Setup Guide)

**ระบบจองห้องไลฟ์สด (Live Studio Booking Management System)**

ระบบนี้เชื่อมโยงหน้าเว็บหลัก (Frontend) เข้ากับฐานข้อมูล Google Sheets ผ่าน Google App Script (GAS) โดยมีขั้นตอนการตั้งค่า 4 ส่วนหลักดังนี้:

---

## ส่วนที่ 1: การตั้งค่า Google Sheets และ Google App Script

เนื่องจากโค้ดฝั่งหลังบ้าน (`Code.gs`) มีระบบสร้างตารางและกรอกข้อมูลเริ่มต้นอัตโนมัติ (Auto-Initialization) คุณจึงไม่จำเป็นต้องพิมพ์หัวตารางเองในแผ่นงาน

1. เข้าไปที่ Google Drive จากนั้นสร้าง **Google Sheets (สเปรดชีต)** ขึ้นมาใหม่ 1 ไฟล์
2. ที่แถบเมนูด้านบน เลือก **ส่วนขยาย (Extensions)** > **Apps Script**
3. ลบโค้ดเริ่มต้นในไฟล์ `รหัส.gs` หรือ `Code.gs` ออกให้หมด
4. คัดลอกโค้ดทั้งหมดจากไฟล์ [Code.gs](file:///Users/raomark/Desktop/WebApp/TH%20Booking/Code.gs) ไปวางแทนที่
5. กดปุ่ม **บันทึกโครงการ (Save)** รูปแผ่นดิสก์ด้านบน

### การทดสอบรันสคริปต์ครั้งแรก (สำคัญ)
1. ในหน้าต่าง Apps Script เลือกฟังก์ชัน `doGet` หรือ `initSpreadsheet` จาก dropdown ด้านบน
2. กดปุ่ม **เรียกใช้ (Run)**
3. ระบบจะขอสิทธิ์การเข้าถึงไฟล์สเปรดชีต (Authorization Required) ให้กด **ตรวจสอบสิทธิ์ (Review Permissions)** เลือกบัญชีกูเกิลของคุณ กด **ขั้นสูง (Advanced)** และคลิกเลือก **ไปที่... (unsafe)** จากนั้นกด **อนุญาต (Allow)**
4. เมื่อรันเสร็จ ให้กลับไปดูที่หน้า Google Sheets จะพบแผ่นงานใหม่ถูกสร้างขึ้นโดยอัตโนมัติ ได้แก่ `Users`, `Rooms`, `Brands`, `Bookings`, `ActivityLog`, `Settings` พร้อมข้อมูลห้อง Room 01 - Room 21 เริ่มต้นเรียบร้อยแล้ว

---

## ส่วนที่ 2: การแผยแพร่ Web App ฝั่งหลังบ้าน (Deployment)

เพื่อให้หน้าเว็บ (Frontend) สามารถส่งคำสั่งมาบันทึกข้อมูลในสเปรดชีตได้ คุณต้องเปิดใช้งาน API ดังนี้:

1. ในหน้าต่าง Apps Script กดปุ่ม **การทำให้ใช้งานได้ (Deploy)** > **การทำให้ใช้งานได้ใหม่ (New deployment)**
2. คลิกรูปเกียร์ (Select type) เลือก **เว็บแอป (Web app)**
3. ตั้งค่ารายละเอียดดังนี้:
   - **คำอธิบาย (Description):** `Live Studio Booking API`
   - **เรียกใช้งานในฐานะ (Execute as):** **ฉัน (บัญชีอีเมลของคุณ)**
   - **ผู้มีสิทธิ์เข้าถึง (Who has access):** **ทุกคน (Anyone)** *(ต้องเลือกหัวข้อนี้ เพื่อให้ Frontend ส่งคำขอแบบ CORS มาหาได้)*
4. กดปุ่ม **การทำให้ใช้งานได้ (Deploy)**
5. รอระบบประมวลผล จากนั้นคัดลอก **URL เว็บแอป (Web app URL)** เก็บไว้ (เช่น `https://script.google.com/macros/s/.../exec`)

---

## ส่วนที่ 3: การสร้าง OAuth Client ID ใน Google Cloud Console (สำหรับ Google Login)

เพื่อใช้งานปุ่มลงชื่อเข้าใช้ด้วยบัญชีกูเกิลอย่างปลอดภัย:

1. เข้าไปที่ [Google Cloud Console](https://console.cloud.google.com/)
2. กดสร้างโครงการใหม่ (**New Project**) และตั้งชื่อโครงการ เช่น `Live Studio Booking`
3. เมนูด้านซ้าย เลือก **APIs & Services** > **OAuth consent screen**
   - เลือกประเภทการใช้งาน (User Type) เป็น **Internal** (สำหรับคนในองค์กร) หรือ **External** (บุคคลทั่วไป) แล้วกดสร้าง
   - กรอกรายละเอียดข้อมูลแอป (App name, Support email, Developer contact email) และกดบันทึกจนเสร็จขั้นตอน
4. ไปที่แถบเมนู **Credentials** (ใบรับรอง)
   - กดปุ่ม **+ Create Credentials** > **OAuth client ID**
   - เลือกประเภทแอปพลิเคชัน (**Application type**) เป็น **เว็บแอปพลิเคชัน (Web application)**
   - ตั้งชื่อใบรับรอง
   - ภายใต้หัวข้อ **Authorized JavaScript origins (ต้นทาง JavaScript ที่ได้รับอนุญาต)** ให้กดเพิ่ม URL ดังนี้:
     - สำหรับทดสอบในเครื่อง: `http://localhost`, `http://127.0.0.1`
     - สำหรับใช้งานจริง: ใส่ URL ของหน้าเว็บที่ Deploy บน Vercel หรือเซิร์ฟเวอร์โฮสติ้งของคุณ
   - กด **สร้าง (Create)**
5. คัดลอกรหัส **Client ID** ที่ได้เก็บไว้ (เช่น `xxxxxxxx-xxxxxx.apps.googleusercontent.com`)

## ส่วนที่ 4: การเชื่อมโยง Frontend เข้ากับระบบหลังบ้าน

1. เปิดไฟล์ [index.html](file:///Users/raomark/Desktop/WebApp/TH%20Booking/index.html) ในโฟลเดอร์โครงการของคุณ
2. ค้นหาข้อความ `data-client_id="YOUR_GOOGLE_CLIENT_ID"` (บรรทึกประมาณ 35) และแก้คำว่า `YOUR_GOOGLE_CLIENT_ID` ให้เป็นรหัส Client ID ที่ได้จาก Google Cloud ในส่วนที่ 3
3. เปิดหน้าเว็บระบบ (ดับเบิลคลิกเปิดไฟล์ `index.html` บนเบราว์เซอร์ของคุณ หรือรันเซิร์ฟเวอร์โลคอล)
4. คลิกที่ **รูปฟันเฟือง (Settings)** มุมล่างซ้ายมือของหน้าจอ
5. วาง **URL เว็บแอป (Web app URL)** ที่คัดลอกไว้จากส่วนที่ 2 ลงในช่องข้อความ แล้วกดตกลง
6. หน้าเว็บจะรีเฟรชตัวเอง และเชื่อมต่อเข้ากับฐานข้อมูล Google Sheets ของคุณทันที!

### ข้อมูลเพิ่มเติมระดับสิทธิ์ในครั้งแรก
- **ผู้ใช้งานคนแรกสุด (First User)** ที่ทำการล็อกอินผ่านปุ่ม Google Sign-in จะได้รับการตั้งค่าเป็นระดับ **Master Admin** โดยอัตโนมัติ เพื่อป้องกันปัญหาระบบล็อก
- เมื่อเข้ามาได้แล้ว แอดมินสามารถเข้าไปที่แท็บ **สิทธิ์ผู้ใช้งาน (Users)** เพื่อเพิ่มอีเมลของทีม Campaign Manager หรือ Viewer รายอื่นๆ เพิ่มเติมได้ตามต้องการ

## ส่วนที่ 5: การตั้งค่าระบบสำรองข้อมูลและล้างประวัติประจำปีอัตโนมัติ (Automated Triggers)

เพื่อให้ระบบทำงานอัปเดตและสำรองข้อมูลอัตโนมัติในเบื้องหลังโดยไม่ต้องมีคนกด:

1. กลับไปที่หน้า Apps Script
2. เมนูด้านซ้าย คลิกไอคอนนาฬิกาปลุก **ทริกเกอร์ (Triggers)**
3. กดปุ่ม **+ เพิ่มทริกเกอร์ (+ Add Trigger)** ด้านล่างขวา
4. ตั้งค่า **ระบบสำรองข้อมูลประจำวัน (Daily Backup)**:
   - เลือกฟังก์ชันที่จะเรียกใช้: `runDailyBackup`
   - เลือกการใช้การผลิตหลัก: `Head`
   - เลือกแหล่งที่มาของกิจกรรม: **ตามเวลา (Time-driven)**
   - เลือกประเภททริกเกอร์ตามเวลา: **ตัวจับเวลาเป็นวัน (Day timer)**
   - เลือกช่วงเวลา: **ตี 1 ถึง ตี 2 (1am to 2am)**
   - กดบันทึก (Save)
5. กดปุ่ม **+ เพิ่มทริกเกอร์** อีกครั้งเพื่อตั้งค่า **ล้างประวัติการจองและย้ายไปตารางสำรองรายปี (Yearly Archive)**:
   - เลือกฟังก์ชันที่จะเรียกใช้: `runYearlyArchive`
   - เลือกการใช้การผลิตหลัก: `Head`
   - เลือกแหล่งที่มาของกิจกรรม: **ตามเวลา (Time-driven)**
   - เลือกประเภททริกเกอร์ตามเวลา: **ตัวจับเวลารายปี (Year timer)**
   - เลือกช่วงเวลา: **31 ธันวาคม (December 31st)** ช่วงเวลา **23:00 ถึง เที่ยงคืน**
   - กดบันทึก (Save)
